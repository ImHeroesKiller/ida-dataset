# Operator Review — UI v1.0

## Operator path (four primary pages + Sources)

1. **Dashboard** — Is learning running? Rows today? Next run?  
2. **Mission** — Start / pause / queue / history  
3. **Sources** — Preferred / trusted / random pool health  
4. **Export** — GitHub / HF / formats status  
5. **Settings** — Confirm cadence, discovery, quality knobs  

## Status language (required)

Use: Learning · Running · Paused · Waiting · Exporting · Publishing · Healthy · Warning · Error · Completed · Idle · Synced  

Never surface: MISCONFIGURED, factory-production, workflow_dispatch, DPS internals, Python stack traces.

## Usability checklist

- [x] Compact type across all pages  
- [x] Consistent card chrome  
- [x] Export simplified  
- [x] Mission simplified  
- [x] Settings grouped  
- [x] Desktop first; tablet/mobile readable  
- [x] Bottom console for live events  
- [x] No empty diagnostic panels  

## Success

Operator manages the factory from Dashboard · Mission · Sources · Export · Settings with one visual language.
