# Learning Contract

## Purpose

Every Directed Learning request becomes a permanent Learning Contract.

## Status: Active (Sprint 4)

## Contract includes

- Mission reference  
- Objective  
- Knowledge scope  
- Priority  
- Allowed sources  
- Resource allocation  
- Deadline  
- Expected deliverables  
- Review requirement  
- Natural language request  

## Lifecycle

1. Human instruction  
2. Mission created (Draft/Queued)  
3. Contract written under `automation/missions/contracts/`  
4. Directed task enters unified learning queue  
5. Scheduler allocates resources  
6. Planner path executes under Policy + Review  
7. Mission report archived under `reports/learning/`  

Contracts are append-only history — they are not deleted when missions complete.
